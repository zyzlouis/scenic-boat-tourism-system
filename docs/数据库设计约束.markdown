**数据库设计约束：**

1. 这个小程序的后台管理将使用微信云开发 CMS 内容管理系统，所以数据库必须对 CMS 友好。
2. 一个集合只存一种事物，集合内所有记录的字段结构必须完全一致。
3. 禁止在一个集合里通过不同 _id 存放结构完全不同的配置（不要把所有配置塞进一个 app_config）。
4. 禁止使用嵌套数组存放列表数据，应该拆分为独立集合，每个元素一条记录。
5. 字段保持扁平，不使用嵌套对象，所有字段都在文档的第一层。
6. 每个集合都要有 sort（排序）、enabled（是否启用）、createdAt、updatedAt 这四个通用字段。
7. 图片类数据不要用数组存在一条记录里，每张图应该是一条独立记录。
8. CMS 支持的字段类型有：单行字符串、多行字符串、数字、布尔值、枚举、图片、文件、富文本、日期。设计字段时请只使用这些类型。

以上，今天就要完成云开发 CMS 内容管理系统的适配，交接给运营人员。
