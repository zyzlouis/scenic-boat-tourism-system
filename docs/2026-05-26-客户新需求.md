## 需求一：项目化商品销售功能

### 业务模型

```
项目（如：水上城堡）
  ├── 商品A：单人儿童票    29.9元  ✅需核销  库存:不限
  ├── 商品B：一大一小亲子票 39.9元  ✅需核销  库存:不限
  ├── 商品C：一大二小亲子票 59.9元  ✅需核销  库存:不限
  ├── 商品D：浴巾          ?元    ❌无需核销  库存:不限
  └── 商品E：防滑袜        ?元    ❌无需核销  库存:不限

未来可新增更多项目（如：游船已有独立流程，不纳入此体系）
```

### 核心规则

| 规则 | 说明 |
|------|------|
| 项目分组 | 商品挂在「项目」下，后台可添加/管理项目 |
| 商品上下架 | 支持上架/下架切换 |
| 是否需要核销 | 商品级别配置。需核销→生成核销码；不需要→购买即完成交易 |
| 核销有效期 | **15 天**，购买即生效 |
| 库存控制 | 默认不限，工作人员可按需设置库存上限 |
| 核销方式 | 一次性核销（员工扫码即完成），与船只的二次核验不同 |

### 后台 admin-web

- 项目管理：添加/编辑/删除项目（名称、图片、描述、营业时间等）
- 商品管理：在项目下添加/编辑/删除/上下架商品
- 商品字段：名称、价格、图片、描述、是否需要核销、库存（0=不限）

### 用户端 client-user

- 首页展示项目和商品（与游船分区展示）
- 商品详情 → 选数量 → 下单购买
- 购买后：需核销的→显示核销码；无需核销的→直接完成
- 我的订单中可查看核销码和状态

### 员工端 client-staff

- 扫码核销商品订单（一次性，不像船只需要进出场）

## 需求二：后台订单数据导出
